# ArUco-навигация: настройка, якоря, препятствия, тест

Камера сверху + ArUco. Калибровка — по **якорям** (4 куба-маркера в известных
местах): система сама строит гомографию каждый кадр, ручной клик пикселей не нужен.
Препятствия — **кубики с маркерами**, детектятся и кладутся в сетку A* на лету.

Всё выполняется **на сервере** (Linux + ROS), где воткнута камера.

---


## 1. Распределение ID маркеров (важно — не пересекать диапазоны)
| Назначение | ID | Роль |
|---|---|---|
| Роботы | 1–9 | поза в реальном времени |
| Якоря (опорные) | 10–19 | строят гомографию; их мировые координаты — в конфиге |
| Препятствия | 20–49 | их клетка добавляется в `blocked` для A* |

Все из одного словаря `DICT_5X5_50`. Генерация листов A4 (из `progs/remote_lab/manager`):
```bash
python navigation/make_markers.py --id 1 --id 10 --id 11 --id 12 --id 13 --id 20 --id 21 --id 22
# -> markers/markers_pageNN.png + markers.pdf (PDF если есть Pillow: pip install pillow)
# раскладывает на A4 с подписями id и контуром реза; печатать 100% / "реальный размер"
```
Полезные флаги: `--marker-mm 50` (размер квадрата), `--per-page 6`, `--gap-mm 12`
(зазор для вырезания), `--dpi 300`. Белую рамку вокруг маркера сохранять.

## 2. Мировая система координат (определить ОДИН раз)
- Начало (0,0) — угол клетки (0,0); +x вдоль `grid_w`, +y вдоль `grid_h`.
- Размер арены = `grid_w*cell_size_m` × `grid_h*cell_size_m`.
- Центр клетки (cx,cy) = ((cx+0.5)*cell_size_m, (cy+0.5)*cell_size_m).
- Курс theta: 0 = смотрит вдоль +x, растёт против часовой, радианы.

Выбери физический угол-ноль и ось +x — от него меряешь координаты якорей.

---

## 3. Якоря — сердце калибровки
- Поставь **4 куба** с маркерами 10–13 по границам зоны (широко, не на одной линии),
  так чтобы они **всегда были в кадре** и робот их не загораживал (по углам).
- Измерь рулеткой мировые (x,y) **центра** каждого якоря, от угла (0,0).
- ⚠️ **Высота:** маркеры якорей должны быть на той же высоте, что маркер робота
  (гомография описывает плоскость на высоте маркеров → иначе параллакс). Маркеры на
  кубах-препятствиях могут быть другой высоты — для препятствий неточность не важна.
- Двигать якоря нельзя без правки конфига: они задают систему координат. А вот
  **камеру двигать можно** — гомография пересчитывается каждый кадр (≥4 якоря видны).

## 4. nav_config.json
Файл `manager/nav_config.json` создаётся из шаблона при первом запуске сервера и
**уже содержит этот пример Теста 1** (его же можно вписать вручную). Пример для арены
1.0 × 0.6 м, клетка 0.1 м (10×6) — подставь свои размеры и измеренные координаты:
```jsonc
{
  "provider": "simulated",        // -> "aruco" после успешного теста (шаг 6)
  "cell_size_m": 0.1,
  "grid_w": 10,
  "grid_h": 6,

  "obstacles": [],                // статические препятствия (клетки), если нужны вручную

  "aruco": {
    "dictionary": "DICT_5X5_50",
    "max_pose_age_s": 0.4,            // поза робота старше -> get_pose() = None
    "obstacle_max_age_s": 3.0,        // куб помнится столько секунд после последнего кадра
    "obstacle_markers": [20, 21, 22], // id кубиков-препятствий
    "cameras": [
      {
        "source": 0,             // индекс USB-камеры (ls /dev/video*) или "/dev/video0"
        "anchors": {             // marker_id -> мировые (x,y) ЦЕНТРА якоря, метры
          "10": [0.05, 0.05],
          "11": [0.95, 0.05],
          "12": [0.95, 0.55],
          "13": [0.05, 0.55]
        }
      }
    ]
  },

  "robots": {
    "Test-device-1": {
      "home": [1, 1],
      "start": [8, 4, 0.0],      // только для симулятора
      "marker_id": 1,
      "marker_yaw_offset": 0.0   // откалибруем на шаге 6
    }
  }
}
```
Две камеры → второй объект в `cameras` со своими `anchors` (свой набор якорей в её
кадре). Препятствия мапятся через ту камеру, что их видит.

## 5. Тест распознавания
Визуально (нужен дисплей или `ssh -X`); сервер при этом НЕ запущен (камеру держит
один процесс):
```bash
python navigation/aruco_test.py --robot Test-device-1
```
- Слева кадр: **синие**=якоря, **красный**=робот, **оранжевые**=кубы-препятствия.
- Справа сетка: робот (стрелка), дом (★), статические (серые) и детектированные
  (оранжевые) препятствия, якоря (синие ×).
- Если в заголовке «anchors N/4» и N<4 — гомографии нет, добавь видимость якорей.

По SSH без дисплея:
```bash
python navigation/aruco_test.py --robot Test-device-1 --headless
# печатает: anchors=4/4  robot: x=.. y=.. th=.. cell=(..)  obstacles=[(..)]
```

### Что проверить
- Робот на сетке там же, где в реальности (подвигай — едет туда же).
- Стрелка курса = «перёд» робота.
- Кубы-препятствия попадают в правильные клетки.
- На краях кадра позиция не уплывает (уплывает → параллакс, проверь высоту якорей).

---

## 6. Калибровка marker_yaw_offset
1. Поставь робота носом вдоль +x.
2. Посмотри `theta` в тесте.
3. Впиши `marker_yaw_offset = -(theta_в_радианах)`, перезапусти тест: вдоль +x → ~0°,
   поворот на +90° CCW → ~90°.

---

## 7. Боевой режим
1. `"provider": "aruco"` в конфиге.
2. `cd progs/remote_lab/manager && python network/server.py`
   В логе: `[aruco] camera 0 opened`, при запуске процедуры —
   `localization = ArucoLocalizationProvider (1 camera(s), DICT_5X5_50)`.
3. Клиентом запусти `all_go_home`. A* объедет статические препятствия + кубы +
   стоящие роботы. Камеры освобождаются при остановке сервера (Ctrl+C).

## Куда что писать
| Что | Где | Единицы |
|---|---|---|
| Размер арены | `grid_w`,`grid_h`,`cell_size_m` | клетки + м/клетка |
| Словарь | `aruco.dictionary` | строка cv2.aruco |
| Камера | `aruco.cameras[i].source` | int или "/dev/videoN" |
| Якоря | `aruco.cameras[i].anchors` | id → (x,y) м центра, на высоте маркера |
| ID препятствий | `aruco.obstacle_markers` | список int |
| Память препятствия | `aruco.obstacle_max_age_s` | секунды |
| id маркера робота | `robots.<name>.marker_id` | int |
| Поворот наклейки | `robots.<name>.marker_yaw_offset` | радианы |
| Цель | `robots.<name>.home` | клетки |

## Типичные проблемы
- `anchors N/4`, N<4 → якоря не видны/перекрыты/не тот id → нет гомографии.
- Маркер не детектится → свет/блики/мелкий/нет белой рамки/не тот словарь.
- Позиция системно смещена → якоря мерили не на высоте маркера (параллакс) или
  перепутаны координаты якорей.
- theta со сдвигом 180° → `marker_yaw_offset = 3.14159`.
- `cannot open camera source 0` → не тот индекс (`--camera 1`) или камеру занял сервер.
